  <name>Task 2: 为 file-preview.tsx 添加 PDF 渲染分支</name>
  <files>frontend/src/components/workspace/preview/file-preview.tsx</files>
  <action>
    在 `file-preview.tsx` 中添加 PDF 文件类型处理，分三步：

    **步骤 1：在 IMAGE_EXTENSIONS 之后添加 PDF_EXTENSIONS 集合和辅助函数**
    ```tsx
    const PDF_EXTENSIONS = new Set(["pdf"]);

    function isPdfFile(ext: string) {
      return PDF_EXTENSIONS.has(ext);
    }
    ```

    **步骤 2：在计算 isImage / isVideo 的逻辑处（约第 147-150 行）添加 isPdf 变量**
    ```tsx
    const isImage = !isCodeFile && isImageFile(ext);
    const isVideo = !isCodeFile && !isImage && isVideoFile(ext);
    const isPdf = !isCodeFile && !isImage && !isVideo && isPdfFile(ext);
    const isPlainText = !isCodeFile && !isImage && !isVideo && !isPdf;
    ```
    注意：`isPdf` 必须排在 `isPlainText` 之前，否则 PDF 仍会落入 plainText 分支。

    **步骤 3：在渲染区域的 isVideo 分支之后、isPlainText fallback 之前插入 isPdf 分支**

    将当前：
    ```tsx
    ) : isVideo ? (
      <VideoPreview src={artifactUrl} filepath={filepath} />
    ) : (
      <pre className="whitespace-pre-wrap p-4 font-mono text-sm">
        {displayPlainText}
      </pre>
    )}
    ```

    修改为：
    ```tsx
    ) : isVideo ? (
      <VideoPreview src={artifactUrl} filepath={filepath} />
    ) : isPdf ? (
      <iframe
        className="size-full"
        src={artifactUrl}
        title={getFileExtension(filepath).toUpperCase() + " preview"}
      />
    ) : isPlainText ? (
      <pre className="whitespace-pre-wrap p-4 font-mono text-sm">
        {displayPlainText}
      </pre>
    ) : (
      <div className="flex size-full items-center justify-center p-4 text-sm text-muted-foreground">
        不支持预览此文件格式
      </div>
    )}
    ```

    说明：
    - `isPdf` 分支使用 `<iframe src={artifactUrl}>` 委托给浏览器内置 PDF 渲染器，与 HTML 预览的实现模式一致
    - 末尾添加 `else` 兜底分支，处理其他无法识别的二进制格式（目前实际上不可达，但提供防御性处理）
    - `isPlainText ? ... : ...` 替代原来的 `else`，结构更清晰
    - 不需要新的 import，`getFileExtension` 已被导入
