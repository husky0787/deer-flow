**hooks.ts**（`"use client"` 指令）：

创建 `useWorkspaceFiles(threadId: string, artifacts: string[])` hook：

1. 从 `./context` import `useWorkspace`。
2. 从 `@/core/uploads` import `useUploadedFiles`（已有 TanStack Query hook）。
3. 用 `useMemo` 合并两个文件来源：
   - `artifacts` 参数（SSE 流中 `thread.values.artifacts`，string[] 绝对路径）→ 映射为 `{ path, source: "artifact" }`
   - `uploadsData?.files` → 映射为 `{ path: f.virtual_path, source: "upload" }`
   - 去重：路径相同时 artifact 优先（先 push artifact，Set 去重）
   - 排序：按 `path` 字母序（`a.path.localeCompare(b.path)`）
4. `useEffect` 将 mergedFiles 同步到 WorkspaceContext 的 `setFiles`。
5. autoSelect 逻辑（`useEffect`）：
   - 检查 `autoSelect` 标志，如果为 false 则 return
   - 检查 `artifacts.length === 0` 则 return
   - 取 `artifacts.at(-1)` 作为最新文件
   - 如果 latest 不等于当前 selectedFile，调用 `selectFile(latest, true)`（isAutoSelect=true）
6. 返回 mergedFiles。

注意 useEffect 依赖数组要精确：`[mergedFiles, setFiles]` 和 `[artifacts, autoSelect, selectedFile, selectFile]`。为避免 ESLint exhaustive-deps 警告，确保所有使用的值都在依赖数组中。如果 `setFiles` 因为不是用 useCallback 包装而导致无限循环，在 context.tsx 中用 useCallback 包装 setFiles、selectFile、deselectFile。
