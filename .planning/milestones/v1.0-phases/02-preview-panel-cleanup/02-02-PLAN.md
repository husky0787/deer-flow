修改 `file-preview.tsx`，添加两个功能：

**1. 大文件截断（PREV-08）**

在文本内容渲染之前添加截断逻辑：
- 定义常量 `const MAX_TEXT_SIZE = 500 * 1024;`（500KB）
- 定义常量 `const TRUNCATE_DISPLAY_CHARS = 50000;`（截断后显示的字符数，约 50K 字符）
- 对 `useArtifactContent` 返回的 `content` 做截断判断：
  ```typescript
  const isTruncated = content != null && content.length > MAX_TEXT_SIZE;
  const displayContent = isTruncated ? content.slice(0, TRUNCATE_DISPLAY_CHARS) : content;
  ```
- 在代码视图（CodeBlock）和纯文本视图（pre）中使用 `displayContent` 替代 `content`
- 在 Markdown/HTML preview 模式中也使用 `displayContent`
- 当 `isTruncated` 时，在内容区域底部显示截断提示横条：
  ```tsx
  {isTruncated && (
    <div className="sticky bottom-0 flex items-center justify-center border-t bg-muted/80 px-4 py-2 text-sm text-muted-foreground backdrop-blur">
      文件过大，仅显示前 {TRUNCATE_DISPLAY_CHARS.toLocaleString()} 字符
    </div>
  )}
  ```

对于纯文本 fallback（非 isCodeFile、非图片/视频），其内容通过独立 fetch 加载。截断逻辑同样适用：
- fetch 完成后判断 `response.text()` 长度是否超过 MAX_TEXT_SIZE
- 如果超过，截取前 TRUNCATE_DISPLAY_CHARS 字符并标记 isTruncated

注意：图片和视频不受截断逻辑影响（它们通过 URL 直接加载，不经过文本内容处理）。

**2. 视频资源释放（PREV-09）**

对视频渲染分支：
- 使用 `useRef<HTMLVideoElement>(null)` 创建 `videoRef`
- 添加 `useEffect` cleanup：
  ```typescript
  useEffect(() => {
    return () => {
      const video = videoRef.current;
      if (video) {
        video.pause();
        video.removeAttribute("src");
        video.load(); // 释放已缓冲的数据
      }
    };
  }, [filepath]); // filepath 变化时释放上一个视频
  ```
- 将 `ref={videoRef}` 绑定到 `<video>` 元素

**结构调整：**
如果视频渲染逻辑和截断逻辑使 FilePreview 组件过长（>150 行），可以将视频预览提取为独立的内部组件 `VideoPreview`（同文件内即可），包含 ref 和 cleanup 逻辑。
